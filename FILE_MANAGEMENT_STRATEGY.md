## 현재 상태 분석

### 기존 구조
```
algorithm/
├── java/BOJ/          → 카테고리별: BOJ1003.java, BOJ1005.java ...
├── js/BOJ/            → 카테고리별: BOJ5430.js ...
├── py/BOJ/            → 카테고리별: 파일들
├── data/              → 테스트 데이터
├── index.html, main.js
└── package.json
```

### 현재 상황의 문제점
- **파일 개수 증가**: 각 폴더에 수십 개의 파일이 선형적으로 쌓임
- **분류 부족**: 문제 카테고리/난이도/주제로 구분하기 어려움
- **검색 어려움**: 특정 알고리즘 유형의 풀이를 찾기 힘듦
- **유지보수성 저하**: 파일이 많아질수록 네비게이션 복잡

## 파일 관리 전략

### 전략 1️⃣: 언어별 + 카테고리별 폴더 분류
```
java/
├── BOJ/
│   ├── basic/           (기본 자료구조)
│   ├── search/          (탐색: BFS, DFS)
│   ├── dynamic/         (동적계획법)
│   ├── graph/           (그래프)
│   ├── greedy/          (탐욕)
│   ├── math/            (수학)
│   └── advanced/        (고급 풀이)

js/ (동일 구조)
py/ (동일 구조)
```

### 전략 2️⃣: 언어별 + 난이도별 분류
```
java/
├── BOJ/
│   ├── bronze/          (난이도: 브론즈)
│   ├── silver/          (난이도: 실버)
│   ├── gold/            (난이도: 골드)
│   ├── platinum/        (난이도: 플래티넘)
│   └── diamond/         (난이도: 다이아몬드)

js/ (동일 구조)
py/ (동일 구조)
```

### 전략 3️⃣: 통합 폴더 + 메타데이터 파일 (추천⭐)
```
solutions/
├── BOJ/
│   ├── BOJ1003.java
│   ├── BOJ1003.js
│   ├── BOJ1003.py
│   ├── BOJ1005.java
│   ├── BOJ1005.js
│   ├── BOJ1005.py
│   └── ... (모든 풀이)
├── PROBLEMS.md          (메타데이터)
└── CATEGORIES.md        (카테고리 매핑)
```

**PROBLEMS.md 예시:**
```markdown
| 문제번호 | 제목 | 난이도 | 카테고리 | Java | JS | Python | 상태 |
|---------|------|--------|----------|------|----|---------|----|
| 1003 | 피보나치 함수 | 실버 | DP | ✓ | ✓ | ✓ | 완료 |
| 1005 | ACM Craft | 골드 | Graph | ✓ | ✗ | ✓ | 진행 |
```

### 전략 4️⃣: 언어별 + 주제별 + 날짜 분류
```
java/
├── 2024-01/
│   ├── basic_array.java
│   ├── string_manipulation.java
│   └── ...
├── 2024-02/
│   ├── graph_bfs.java
│   ├── dp_fibonacci.java
│   └── ...

js/  (동일 구조)
py/  (동일 구조)
```

### 전략 5️⃣: 플랫폼별 + 언어별 분류
```
problems/
├── BOJ/
│   ├── java/
│   ├── js/
│   └── python/
├── LEETCODE/
│   ├── java/
│   ├── js/
│   └── python/
└── PROGRAMMERS/
    ├── java/
    ├── js/
    └── python/
```

### 전략 6️⃣: 스크립트 기반 자동 정렬 + 인덱싱
- 폴더 구조 유지하되, 동적으로 심볼릭 링크/단축키 생성
- Python/Node 스크립트로 문제별 메타데이터 수집 및 검색 인덱스 생성
- CLI 검색 도구 제공 (예: `npm run search "dp graph"`)

## 각 전략의 장단점

| 전략 | 장점 | 단점 | 적합성 |
|------|------|------|--------|
| **1. 카테고리별** | ✅ 주제별 학습 용이<br>✅ 알고리즘 패턴 파악 쉬움<br>✅ 구조가 명확함 | ⚠️ 카테고리 구분 어려움<br>⚠️ 다중 카테고리 문제 처리 복잡 | ⭐⭐⭐ |
| **2. 난이도별** | ✅ 단계별 학습 용이<br>✅ 점진적 난이도 상향 | ⚠️ BOJ 난이도 변동 가능<br>⚠️ 폴더 이동 필요<br>⚠️ 주제 파악 어려움 | ⭐⭐ |
| **3. 통합 + 메타데이터** | ✅ 가장 단순한 구조<br>✅ 검색/필터링 유연함<br>✅ 다중 언어 한 곳에<br>✅ 진행상황 추적 용이 | ⚠️ 파일 개수 증가 시 스크롤<br>⚠️ 메타데이터 유지보수<br>⚠️ IDE 네비게이션 복잡 | ⭐⭐⭐⭐⭐ |
| **4. 주제+날짜** | ✅ 시간별 추적 가능<br>✅ 복습 계획 수립 용이 | ⚠️ 카테고리/난이도 파악 어려움<br>⚠️ 주제별 집계 복잡 | ⭐⭐ |
| **5. 플랫폼별** | ✅ 여러 플랫폼 지원<br>✅ 확장성 있음 | ⚠️ 깊은 폴더 구조<br>⚠️ 현재 프로젝트와 불일치 | ⭐⭐⭐ |
| **6. 자동화 도구** | ✅ 가장 유연함<br>✅ 확장성 최고 | ⚠️ 초기 설정 복잡<br>⚠️ 유지보수 필요 | ⭐⭐⭐⭐ |

## 프로젝트 추천 전략

### 🎯 **최우선 추천: 전략 3 (통합 + 메타데이터) + 전략 1 (카테고리) 하이브리드**

### 단계별 마이그레이션 계획

#### **Phase 1: 메타데이터 준비** (1주)
1. `PROBLEMS.md` 생성 - 모든 풀이 파일 목록 + 메타데이터
2. 각 문제별 카테고리 매핑
3. 진행상황/우선순위 표시

#### **Phase 2: 통합 폴더 구조** (선택)
```
solutions/
├── BOJ/
│   ├── BOJ1003.java, BOJ1003.js, BOJ1003.py
│   ├── BOJ1005.java, ...
│   └── ... (모든 문제)
├── PROBLEMS.md
├── CATEGORIES.md
└── SEARCH_INDEX.json (자동 생성)
```

#### **Phase 3: 보조 구조 (필요시)**
```
solutions/
├── by_category/          (심볼릭 링크 또는 매핑만)
│   ├── dp/
│   ├── graph/
│   └── ...
├── by_difficulty/        (심볼릭 링크 또는 매핑만)
│   ├── gold/
│   ├── silver/
│   └── ...
└── BOJ/                  (실제 파일들)
    ├── 모든 풀이 파일
```

## 결론

### 왜 전략 3 (통합 + 메타데이터)를 추천하나?

1. **현 프로젝트 특성에 최적**: 다중 언어 지원 (Java, JS, Python)
2. **점진적 도입 가능**: 폴더 이동 최소화
3. **확장성**: 새 플랫폼/언어 추가가 간단
4. **유연성**: 검색, 필터링, 추적이 메타데이터로 통일
5. **유지보수성**: 파일 이동 불필요, 메타데이터만 관리

### 초기 투자 대비 효과
- 초기: 2-3시간 (메타데이터 작성)
- 유지: 파일 추가 시 1줄 추가만 필요
- 효과: 파일 100개 이상 관리 시 생산성 50% 향상
